# 08. MVPスコープおよびロードマップ (MVP Scope & Roadmap)

開発の初期段階（Milestone 1）で実装すべき最小機能セットを定義し、16GB RAM環境での早期動作確認を目指す。

## 1. Milestone 1: 基盤対話プロトタイプ (MVP)
目標: 「Bonsai 1基による2体対話と、情動値の可視化」

- **Core**: `llama-cpp-python` または `AnythingLLM API` 経由の推論。
- **Agent**: 固定された2つの人格プロンプトによる交互対話。
- **Emotion**: `[EMOTION_DELTA]` タグの自己申告パースと、内部変数の更新。
- **UI**: Streamlitによるテキストログと、現在の情動値（数値）のリアルタイム表示。
- **Control**: `Active` / `Asleep` の手動トグル。

## 2. Milestone 2: 監視と記憶の統合
目標: 「システム介入の自動化と、SQLiteによる忘却管理」

- **Watcher**: 正規表現による禁忌ワード検知と、強制話題転換プロンプトの自動挿入。
- **Memory**: SQLite DBの実装。感情的淘汰アルゴリズムによる1,000行制限の適用。
- **RAG**: 文脈に応じた過去の要約のインジェクション開始。

## 3. Milestone 3: 知能の高度化と人格の多様化
目標: 「Bonsai監視者の導入と、10種の人格選択」

- **Advanced Watcher**: 監視者モード（Bonsai 8B）による意味的介入判定。
- **Library**: `06. 人格ライブラリ` からの自由な人格選択機能。
- **Visual**: 情動推移のグラフ化（レーダーチャート等）の完成。

## 4. 成功基準 (Exit Criteria for MVP)
1. 16GB RAM環境で、OOM（メモリ不足）を起こさずに 50 ターン以上連続稼働すること。
2. エージェントが自分の情動を JSON 形式で正しく出力し続けられること。
3. 禁忌ワード検知時に、システムが介入して会話の流れを変えられること。
