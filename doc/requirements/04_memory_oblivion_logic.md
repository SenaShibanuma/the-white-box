# 04. 記憶永続化と忘却ロジック要件 (Memory Persistence & Oblivion Logic)

## 1. コンテキスト窓の厳格な管理 (RAM Protection)

16GB RAMを保護するため、LLMの作業領域（コンテキスト窓）のオーバーフローを物理的に防ぐ。

- **スライディング・ウィンドウ**: コンテキストが閾値（例: 4,000トークン）を超過した場合、最も古い対話ブロックをRAMから即座に破棄する。
- **再帰的要約**: 破棄の直前、Qwen 0.6Bを用いて該当ブロックを「1〜2行の要約（事実と感情の記録）」に圧縮する。

## 2. 記憶のゲートキーパーロジック（SSD退避前処理）

無価値なループ会話によるデータベース膨張を防ぐためのフィルタリング機構。

- **感情デルタ・フィルタ**:
  要約ブロック内の感情変動幅（$\Delta D, \Delta F$ など）を計算する。変動が極めて小さい「退屈な雑談」は、SSDに保存せずRAMから**即時揮発（破棄）**させる。
- **SSD書き込み**:
  閾値を超えた「強い情動を伴う記憶」のみ、SQLiteデータベース（SSD）に、感情タグおよびタイムスタンプと共に永続化する。

## 3. 記憶の膨張対策ロジック：ハードリミット

長期稼働においてもSQLite DBのサイズを最小限に保つための絶対防壁。

- **レコード上限**: データベースの最大保持数を `1,000行` に固定する。
- **感情的淘汰（Affective Eviction）**:
  1,001行目の記憶が挿入されようとした際、単に古いものを消すのではなく、SSD内の全レコードのうち**「感情タグの絶対値（喜怒哀楽の強さ）が最も低い記憶」**を検索し、それを `DELETE` する。
- **効果**: エージェントの長期記憶には、平坦な日常は残らず、極度の恐怖（Friction）や特別な安堵（Resonance）だけが純度高く抽出されていく。

## 4. 完全忘却と想起ロジック

- **不可逆の忘却**: 感情적淘汰によってSQLiteから `DELETE` されたデータは、システム上二度と復元できない。エージェントは「具体的な事象」を完全に失う。
- **オンデマンド想起 (RAG)**:
  現在の対話キーワード、または現在の極端な感情ステータスに合致する過去の要約のみをSQLiteから検索し、直近のコンテキストに挿入する。
